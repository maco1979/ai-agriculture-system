## 研究总结

### 1. Azure Static Web Apps
- **免费层**: 完全免费，100GB带宽/月
- **地区限制**: 有限地区支持，`eastasia` (东亚) 可用
- **构建配置**: React 应用输出到 `build` 目录
- **部署**: 自动从 GitHub 或 Azure DevOps 部署

### 2. Azure Container Apps
- **免费额度**: 每月180,000 vCPU秒 + 360,000 GiB内存秒
- **消费计划**: 按使用付费，适合POC
- **必需资源**: Container Apps Environment, Container Registry
- **监控**: 需要 Log Analytics 和 Application Insights

### 3. Supabase 免费层
- **数据库**: 500MB PostgreSQL + 50,000行插入/月
- **认证**: 提供连接字符串和API密钥
- **限制**: 适合POC测试，生产环境需升级
- **数据安全**: 数据可能不在中国境内

### 4. Application Insights
- **免费层**: 每月5GB数据摄入
- **监控**: 应用性能监控和日志收集
- **集成**: 与 Container Apps 和 Static Web Apps 集成
- **建议**: 调用 `appinsights-instrumentation` 技能进行配置

### 5. 地区可用性注意事项
- **Static Web Apps**: 在 `eastasia` 地区可用（包括中国香港）
- **Container Apps**: 在中国东部2等主要地区可用
- **Supabase**: 全球服务，可能不在中国境内
- **建议地区**: 中国东部2 (eastchina2) 或东亚 (eastasia)

### 6. 资源命名规则
- 所有资源名称必须符合 Azure 命名规则
- 使用小写字母、数字和连字符
- 避免特殊字符和空格
- 保持名称简洁且有描述性
