# 自动学习控制本地硬件系统 - 使用指南

## 系统概述

自动学习控制本地硬件系统是一个智能化的硬件管理平台，集成了以下核心功能：

- **硬件自动检测**: 自动扫描和识别本地网络、蓝牙、红外设备
- **智能决策**: 基于AI智能体的多维度决策系统
- **自动控制**: 支持多种控制策略（反应式、预测式、自适应、最优控制）
- **持续学习**: 通过机器学习不断优化控制参数和策略

## 快速开始

### 1. 运行演示模式

```bash
# 运行60秒自动演示
python run_auto_learn_hardware.py

# 运行120秒演示
python run_auto_learn_hardware.py -d 120
```

### 2. 交互模式

```bash
# 启动交互式命令行
python run_auto_learn_hardware.py -i
```

### 3. 编程接口

```python
import asyncio
from auto_learn_hardware_control import AutoLearnHardwareControlSystem

async def main():
    # 创建系统实例
    system = AutoLearnHardwareControlSystem()
    
    # 初始化系统
    await system.initialize_system()
    
    # 手动控制设备
    result = await system.manual_control(
        device_id="123",
        action="cool_down",
        parameters={"intensity": 0.8}
    )
    
    # 查看系统状态
    status = system.get_system_status()
    print(status)
    
    # 清理资源
    await system.cleanup()

asyncio.run(main())
```

## 功能特性

### 1. 硬件自动检测

系统支持自动检测以下类型的设备：

- **网络设备**: 通过IP扫描发现Web服务器、SSH设备、Windows设备等
- **蓝牙设备**: 温湿度传感器、土壤传感器、光照传感器、控制器等
- **红外设备**: 红外控制器、遥控设备等
- **摄像头**: 本地摄像头，支持真实和模拟模式

### 2. 控制策略

系统支持四种控制策略：

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| Reactive | 反应式控制 | 快速响应环境变化 |
| Predictive | 预测式控制 | 基于历史数据预测 |
| Adaptive | 自适应控制 | 根据反馈自动调整 |
| Optimal | 最优控制 | 追求最优性能指标 |

### 3. 学习机制

自适应学习引擎包含以下功能：

- **动作记录**: 记录每次控制动作及其结果
- **成功模式识别**: 学习成功的控制参数组合
- **失败模式分析**: 避免重复失败的控制策略
- **参数优化**: 自动优化控制参数
- **成功预测**: 预测控制动作的成功概率

### 4. 支持的控制类型

- **温度控制**: `cool_down` - 自动降温
- **功耗优化**: `optimize_power` - 优化能源使用
- **光谱调整**: `adjust_spectrum` - 调整LED光谱
- **摄像头控制**: `camera_capture` - 图像捕获
- **蓝牙控制**: `bluetooth_command` - 蓝牙设备控制
- **红外控制**: `infrared_command` - 红外设备控制

## 命令行使用


```bash
  -h, --help       显示帮助信息
```

## 系统状态说明

### 学习状态字段

| 字段 | 说明 |
|------|------|
| total_decisions | 总决策次数 |
| successful_controls | 成功控制次数 |
| failed_controls | 失败控制次数 |
| learning_cycles | 学习周期数 |
| performance_score | 性能分数 (0-1) |

### 设备状态字段

| 字段 | 说明 |
|------|------|
| status | 设备状态 (online/offline) |
| connected | 是否已连接 |
| last_control | 最后控制时间 |
| control_count | 控制次数 |
| success_count | 成功次数 |

## 配置文件

系统配置文件 `auto_learn_hardware_config.json` 包含以下设置：

### 学习参数

```json
{
  "learning": {
    "learning_rate": 0.1,        // 学习率
    "learning_interval": 10,     // 学习间隔（秒）
    "decision_interval": 2,      // 决策间隔（秒）
    "control_interval": 1        // 控制间隔（秒）
  }
}
```

### 硬件配置

```json
{
  "hardware": {
    "spectrum": {
      "serial_port": "/dev/ttyUSB0",
      "baud_rate": 9600,
      "led_count": 20000
    },
    "camera": {
      "default_index": 0,
      "resolution": {"width": 640, "height": 480}
    }
  }
}
```

## API接口

系统提供RESTful API接口：

### 获取设备列表
```http
GET /ai-control/devices
```

### 控制设备
```http
POST /ai-control/device/{id}
Content-Type: application/json

{
  "action": "cool_down",
  "parameters": {"intensity": 0.8}
}
```

### 获取主控状态
```http
GET /ai-control/master-control/status
```

### 激活/关闭主控
```http
POST /ai-control/master-control
Content-Type: application/json

{
  "activate": true
}
```

## 故障排除

### 常见问题

1. **设备无法检测**
   - 检查网络连接
   - 确认蓝牙/红外适配器正常工作
   - 查看日志文件 `auto_learn_hardware.log`

2. **控制命令失败**
   - 检查设备是否在线
   - 确认设备权限设置
   - 查看学习引擎的成功预测分数

3. **学习效果不佳**
   - 增加学习周期数
   - 调整学习率参数
   - 检查数据质量

### 日志查看

```bash
# 实时查看日志
tail -f auto_learn_hardware.log

# 查看错误信息
grep "ERROR" auto_learn_hardware.log
```
